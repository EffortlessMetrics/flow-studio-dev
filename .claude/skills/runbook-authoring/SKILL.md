# Runbook Authoring

1. Define purpose (what this accomplishes, 1-2 lines).
2. List prerequisites (what must be true before starting).
3. Write steps: numbered, one action per step, imperative.
4. Add verification: commands + expected signals for each step.
5. Document rollback (how to undo safely).
6. Include troubleshooting (common issues and fixes).
